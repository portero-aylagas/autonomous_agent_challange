The hardest part of planning was keeping the agent narrow enough to feel realistic without turning it into a full defect-management platform. If I did this again, I would define the evaluation dataset and review rubric earlier so the success metrics were less abstract from the start. The biggest open question is how much domain-specific context, such as prior defect patterns or UI taxonomies, the agent would need before its suggested reproduction steps and regression scenarios become consistently trustworthy for a real QA team.
